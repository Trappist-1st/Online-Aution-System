# 基于 Java RMI 的在线拍卖系统 - 项目文档

## 一、项目概述

### 1.1 项目名称
在线拍卖系统（Online Auction System）

### 1.2 项目简介
本项目是一个基于 Java RMI（Remote Method Invocation）分布式通信机制实现的在线拍卖系统。系统采用客户端-服务器架构，允许多个买家（客户端）通过拍卖中心（服务器）对商品进行竞价，实现实时的分布式拍卖功能。

### 1.3 项目定位
- **复杂度**：中等，满足实验要求的基础功能
- **扩展性**：预留有限扩展空间（如多商品拍卖、用户认证等）
- **适用场景**：分布式系统课程实验、RMI 技术学习演示

## 二、需求分析

### 2.1 功能性需求

#### 2.1.1 服务器端（拍卖中心）功能
1. **拍卖物品管理**
   - 初始化拍卖物品（物品名称、起拍价）
   - 管理拍卖物品的状态（进行中/已结束）

2. **出价管理**
   - 接收客户端的出价请求
   - 验证出价是否有效（高于当前最高价）
   - 记录最高出价和买家信息

3. **实时通知**
   - 当有新的更高出价时，通知所有注册的客户端
   - 推送当前最高出价和买家姓名

4. **客户端管理**
   - 维护已注册客户端列表
   - 处理客户端注册和注销

#### 2.1.2 客户端（买家）功能
1. **查看拍卖信息**
   - 显示当前拍卖的物品名称
   - 显示当前最高出价
   - 显示当前最高出价者姓名

2. **参与竞拍**
   - 输入买家姓名
   - 输入出价金额
   - 提交出价到服务器

3. **接收实时更新**
   - 接收服务器推送的最高出价更新
   - 实时刷新界面显示

### 2.2 非功能性需求

1. **性能要求**
   - 支持至少 10 个并发客户端
   - 出价响应时间 < 1 秒

2. **可用性要求**
   - 简洁的命令行界面
   - 清晰的操作提示

3. **可靠性要求**
   - 出价数据不丢失
   - 客户端异常断开不影响服务器运行

4. **可扩展性**
   - 代码结构清晰，便于功能扩展
   - 接口设计灵活

## 四、系统架构设计

### 4.1 整体架构

系统采用 **三层架构**：

1. **表示层（Presentation Layer）**
   - 客户端用户界面（命令行）
   - 负责用户交互和信息展示

2. **业务逻辑层（Business Logic Layer）**
   - 服务器端拍卖逻辑
   - 出价验证、状态管理

3. **通信层（Communication Layer）**
   - RMI 远程接口
   - RMI Registry 服务注册

### 4.2 核心组件

#### 4.2.1 服务器端组件

1. **AuctionServer（拍卖服务器）**
   - 主类，启动 RMI 服务
   - 初始化 RMI Registry

2. **AuctionServiceImpl（拍卖服务实现）**
   - 实现拍卖业务逻辑
   - 管理拍卖物品和出价记录
   - 维护客户端回调列表

3. **AuctionItem（拍卖物品）**
   - 物品名称、起拍价
   - 当前最高价、最高出价者

#### 4.2.2 客户端组件

1. **AuctionClient（拍卖客户端）**
   - 主类，启动客户端程序
   - 连接到 RMI Registry

2. **ClientCallbackImpl（客户端回调实现）**
   - 接收服务器推送的更新
   - 刷新本地显示

3. **ClientUI（客户端界面）**
   - 命令行界面
   - 用户输入处理

## 五、接口设计

### 5.1 远程接口定义

#### 5.1.1 AuctionService（服务器端远程接口）

```java
public interface AuctionService extends Remote {
    // 获取当前拍卖物品信息
    AuctionItem getAuctionItem() throws RemoteException;
    
    // 提交出价
    boolean placeBid(String bidderName, double amount) throws RemoteException;
    
    // 注册客户端回调
    void registerClient(ClientCallback callback) throws RemoteException;
    
    // 注销客户端回调
    void unregisterClient(ClientCallback callback) throws RemoteException;
}
```

#### 5.1.2 ClientCallback（客户端回调接口）

```java
public interface ClientCallback extends Remote {
    // 接收拍卖更新通知
    void notifyBidUpdate(String itemName, double currentPrice, 
                        String currentBidder) throws RemoteException;
}
```

### 5.2 数据模型

#### 5.2.1 AuctionItem（拍卖物品类）

```java
public class AuctionItem implements Serializable {
    private String itemName;        // 物品名称
    private double startingPrice;   // 起拍价
    private double currentPrice;    // 当前最高价
    private String currentBidder;   // 当前最高出价者
    private boolean isActive;       // 是否进行中
}
```

## 六、详细设计

### 6.1 服务器端流程

#### 6.1.1 服务器启动流程
1. 创建并启动 RMI Registry（端口 1099）
2. 创建 AuctionServiceImpl 实例
3. 将服务绑定到 RMI Registry
4. 初始化拍卖物品
5. 等待客户端连接

#### 6.1.2 出价处理流程
```
客户端提交出价
    ↓
验证出价金额 > 当前最高价？
    ↓ 是
更新最高价和出价者信息
    ↓
遍历所有注册的客户端
    ↓
通过回调接口通知所有客户端
    ↓
返回成功
```

### 6.2 客户端流程

#### 6.2.1 客户端启动流程
1. 连接到 RMI Registry
2. 查找 AuctionService
3. 创建 ClientCallback 实例
4. 注册回调到服务器
5. 获取并显示当前拍卖信息
6. 等待用户输入

#### 6.2.2 出价流程
```
用户输入姓名和出价
    ↓
调用 placeBid() 远程方法
    ↓
接收返回结果
    ↓
显示成功或失败消息
    ↓
等待服务器推送更新
```

## 十二、常见问题与解决方案

### 12.1 RMI 相关问题

**Q1: java.rmi.NotBoundException**
- **原因**：客户端无法找到服务器绑定的服务
- **解决**：确保服务器已启动且服务已绑定

**Q2: java.rmi.ConnectException**
- **原因**：无法连接到 RMI Registry
- **解决**：检查服务器 IP 和端口，确认防火墙设置

**Q3: java.rmi.UnmarshalException**
- **原因**：客户端和服务器的类版本不一致
- **解决**：确保使用相同的代码版本

### 12.2 并发问题

**Q4: 多个客户端同时出价导致数据不一致**
- **解决**：在 placeBid() 方法上添加 synchronized 关键字

### 12.3 网络问题

**Q5: 局域网测试时客户端连接不上**
- **解决**：
  1. 确认服务器 IP 地址
  2. 关闭防火墙或添加例外规则
  3. 使用 `java.rmi.server.hostname` 系统属性指定服务器 IP

## 十三、学习要点

### 13.1 RMI 核心概念

1. **远程接口**：必须继承 `java.rmi.Remote`，所有方法必须抛出 `RemoteException`
2. **远程对象**：实现远程接口的类，通常继承 `UnicastRemoteObject`
3. **RMI Registry**：服务注册与发现中心
4. **Stub 和 Skeleton**：客户端代理和服务器端骨架（Java 5+ 自动生成）

### 13.2 关键技术点

1. **RMI 回调机制**：实现服务器向客户端推送消息
2. **序列化**：远程传输的对象必须实现 `Serializable`
3. **异常处理**：正确处理 `RemoteException`
4. **线程安全**：服务器端的并发控制

### 13.3 实验收获

- 理解分布式系统的基本架构
- 掌握 RMI 的使用方法
- 学习客户端-服务器通信模式
- 体验分布式系统的挑战（网络延迟、并发、异常处理）
